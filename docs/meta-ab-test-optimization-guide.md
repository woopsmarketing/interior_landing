# Meta 광고 A/B 테스트 최적화 가이드

> 작성일: 2026-04-13
> 현재 상태: 캠페인 1개 / 광고세트 2개 (스토리+릴스, 피드) / 전환 0건
> 목적: 공정한 소재 비교 → 승자 확정 → 폼 A/B 테스트 → 전환 캠페인 전환

---

## 현재 구조 및 문제점

### 현재 구조
```
캠페인 1개 (트래픽 / 랜딩 페이지 조회)
├─ 광고세트: 스토리+릴스 (일일 ₩10,000)
│   ├─ S1_숨겨진정보    →  26 노출, 1 결과
│   ├─ S2_가격충격      → 217 노출, 14 결과 ⭐ CTR 6.45%
│   ├─ S3_비교견적      →  39 노출, 2 결과
│   ├─ S4_손해불안      → 197 노출, 8 결과 ⭐ CTR 4.06%
│   ├─ S5_공감텍스트    → 930 노출, 18 결과 ← 노출 독식 / CTR 1.94%
│   └─ S6_사회적증거    →   0 노출, 0 결과 ← 완전히 죽음
│
└─ 광고세트: 피드 (일일 ₩10,000)
    ├─ F1_궁금증_1600만원 →  9 노출, 0 결과
    ├─ F2_숨겨진정보      → 43 노출, 0 결과
    ├─ F3_견적차이        →  2 노출, 0 결과
    ├─ F4_가격충격        →  6 노출, 0 결과
    └─ F5_AB비교          → 238 노출, 9 결과 ← 유일한 성과
```

### 핵심 문제
1. **불공정한 노출 분배**: 한 광고세트에 여러 소재 → Meta가 Multi-Armed Bandit 알고리즘으로 승자 독식
2. **S5가 노출 독식**: 930/1,409 노출(66%)을 가져갔지만 CTR은 최하 1.94%
3. **피드 4개 소재 사실상 사망**: F5만 노출, 나머지는 기회조차 못 받음
4. **예산 소진 불량**: 일일 ₩20,000 중 ~₩11,000만 사용 (55%)

## Phase 1: 소재 A/B 테스트 (1주일)

### 목표
모든 소재에 균등한 기회를 주고, 공정하게 CTR을 비교하여 승자를 확정한다.

### 예산: 주 20~30만원 기준 (일일 ₩30,000~40,000)

### 추천 구조 (일일 ₩40,000 / 주 ₩280,000)

```
[기존 캠페인] → 모든 광고세트 OFF (삭제하지 말 것, 데이터 보존)

[새 캠페인] 소재 테스트 (A/B 테스트 ON)
├─ 광고세트 S2: S2_가격충격만      일일 ₩10,000  ← 최고 CTR 6.45%
├─ 광고세트 S4: S4_손해불안만      일일 ₩10,000  ← 2등 CTR 4.06%
├─ 광고세트 S5: S5_공감텍스트만    일일 ₩10,000  ← 노출 독식, 재검증
└─ 광고세트 F5: F5_AB비교만        일일 ₩10,000  ← 피드 유일 성과

총 일일 예산: ₩40,000
기간: 7일
총 비용: ~₩280,000
```

### 절약 옵션 (일일 ₩30,000 / 주 ₩210,000)

```
├─ 광고세트 S2: S2_가격충격만      일일 ₩10,000
├─ 광고세트 S4: S4_손해불안만      일일 ₩10,000
└─ 광고세트 F5: F5_AB비교만        일일 ₩10,000
```

### 나머지 소재 (S1, S3, S6, F1~F4)는?
Phase 1 승자 확정 후 별도 테스트 가능.
예산 한도 내에서 소재당 ₩10,000 유지가 핵심.

### ⚠️ 주의: 소재당 최소 ₩10,000/일 필요
₩10,000 미만으로 내리면 하루에 50~100노출밖에 안 됨 → 7일 해도 350~700노출 → 통계적 유의성 부족.
소재 수를 줄이더라도 소재당 ₩10,000은 유지할 것.

## Meta Ads Manager에서 수정하는 방법 (단계별)

### Step 1: 기존 캠페인 OFF

```
1. Meta Ads Manager (ads.manager.facebook.com) 접속
2. 왼쪽 메뉴 → "모든 캠페인"
3. 기존 캠페인 이름 옆 토글 → OFF (비활성화)
   ⚠️ 삭제하지 말 것! 기존 데이터 보존 필요
```

### Step 2: 새 캠페인 만들기 (스토리/릴스용)

```
1. 좌측 상단 "+ 만들기" 클릭
2. 캠페인 설정:
   ┌────────────────────────────────────────┐
   │ 구매 유형:        경매                  │
   │ 캠페인 목표:      트래픽                │
   │ 캠페인 이름:      스토리_소재테스트_v2   │
   │ A/B 테스트:       ✅ ON (활성화!)       │
   │ 예산 전략:        광고 세트 예산         │
   │ 캠페인 입찰 전략:  최고 볼륨             │
   └────────────────────────────────────────┘
3. "계속" 클릭
```

### Step 3: A/B 테스트 설정

```
A/B 테스트를 ON 하면 추가 설정이 나타남:

┌────────────────────────────────────────┐
│ 테스트 변수:  "크리에이티브" 선택        │ ← 소재를 비교하는 것이므로
│ 버전 수:     6 (S1~S6)                 │
│ 테스트 기간:  7일                       │
│ 성공 기준:   "랜딩 페이지 조회당 비용"   │ ← CPC 기준 비교
│ 신뢰 수준:   95% (기본값 유지)          │
└────────────────────────────────────────┘

⚠️ 만약 "테스트 변수"에서 "크리에이티브"를 선택할 수 없다면:
→ A/B 테스트를 OFF 하고, 대신 광고세트를 6개 수동으로 만들기 (아래 Step 4 참고)
→ 수동으로 만들어도 각 광고세트가 독립 예산이므로 균등 노출 됨
```

### Step 4: 광고세트 6개 만들기 (각 소재별)

**모든 광고세트 공통 설정:**

```
┌────────────────────────────────────────────┐
│ 트래픽 전환 위치:  웹사이트                  │
│ 성과 목표:        랜딩 페이지 조회 수 최대화  │
│ 일일 예산:        ₩10,000                   │
│ 시작/종료 날짜:   시작일 ~ +7일              │
│ 타겟:                                       │
│   위치:           대한민국                    │
│   연령:           25~55세 (변경 권장, 아래 참고) │
│   성별:           전체                       │
│   관심사:         설정 안 함 (Broad 타겟)     │
│ 노출 위치:                                  │
│   플랫폼:         Instagram만               │
│   배치:           스토리 + Reels             │
│ 최적화 및 게재:                              │
│   입찰 전략:      최고 볼륨 (기본값)          │
└────────────────────────────────────────────┘
```

**⚠️ 연령대 변경 권장: 18세 → 25~55세**
- 18~24세는 인테리어 수요가 거의 없음 (전세/자가 비율 낮음)
- 광고비를 실제 수요층에 집중하기 위해 25세 이상으로 변경
- 55세 이상은 인스타그램 사용률 낮으므로 55세까지 권장

**광고세트별 설정:**

| 광고세트 이름 | 소재 | 랜딩 URL |
|-------------|------|---------|
| `S1_숨겨진정보` | S1 이미지 1개만 | `도메인/form?utm_source=instagram&utm_medium=story&utm_content=s1` |
| `S2_가격충격` | S2 이미지 1개만 | `도메인/form?utm_source=instagram&utm_medium=story&utm_content=s2` |
| `S3_비교견적` | S3 이미지 1개만 | `도메인/form?utm_source=instagram&utm_medium=story&utm_content=s3` |
| `S4_손해불안` | S4 이미지 1개만 | `도메인/form?utm_source=instagram&utm_medium=story&utm_content=s4` |

### Step 5: 피드 캠페인도 동일하게 생성

```
캠페인 이름: 피드_소재테스트_v2

5개 광고세트 (F1~F5) 동일 방식으로 생성
차이점:
  - 노출 위치: Instagram 피드 + 탐색탭

```

```
│ 행동 유도:     "더 알아보기" 또는 "신청하기" │
│ 웹사이트 URL:  위 표의 랜딩 URL         │
│ URL 매개변수:  자동 (위 URL에 이미 포함)  │
└────────────────────────────────────────┘
```

## Phase 2: 폼 A/B 테스트 (Phase 1 승자 확정 후)

### 전제조건
- Phase 1에서 스토리 승자 2~3개, 피드 승자 1~2개 확정
- /form-v2 (3스텝 축소 폼) 개발 완료

### 구조

```
[캠페인] 폼_AB테스트
├─ 광고세트 A: 승자소재 → /form (현재 5스텝)     일일 ₩15,000
└─ 광고세트 B: 승자소재 → /form-v2 (새 3스텝)    일일 ₩15,000

→ 동일 소재, 동일 타겟, 랜딩 URL만 다름
→ 순수하게 폼 전환율만 비교 가능
```

### 설정 방법

```
1. 새 캠페인 만들기 → A/B 테스트 ON
2. 테스트 변수: "타겟" (사실은 랜딩 URL이 다른 것이지만,
   크리에이티브는 동일하므로 광고세트 단위로 분리)
3. 광고세트 A: 랜딩 URL → 도메인/form?utm_source=...&utm_campaign=form_v1
4. 광고세트 B: 랜딩 URL → 도메인/form-v2?utm_source=...&utm_campaign=form_v2
5. 나머지 설정(타겟, 예산, 배치)은 모두 동일
6. 테스트 기간: 7~14일 (전환 데이터 필요하므로 더 길게)
7. 성공 기준: "전환" (Meta Pixel Lead 이벤트)
```

## Phase 3: 전환 캠페인 전환 (전환 1건+ 확보 후)

### 전제조건
- Meta Pixel 설치 완료
- 폼 제출 전환 이벤트 1건 이상 기록

### 변경 사항

```
기존: 캠페인 목표 "트래픽" → 변경: "전환" 또는 "리드"

1. 새 캠페인 만들기
2. 캠페인 목표: "리드" (리드 생성)
3. 전환 위치: 웹사이트
4. 전환 이벤트: Lead (폼 제출 완료)
5. 승자 소재 + 승자 폼 URL 사용
6. 일일 예산: ₩30,000~50,000 (전환 캠페인은 더 많은 예산 필요)
```

## 타겟 오디언스 최적화

### 현재 설정 (문제)
```
위치:    대한민국 (OK)
연령:    18세 이상 (너무 넓음)
관심사:  없음 (너무 넓음)
```

### 권장 설정
```
위치:    대한민국
연령:    25~55세
성별:    전체
관심사:  설정 안 함 (Broad로 Meta AI에 맡기기)

※ 관심사를 좁히면 도달이 줄어서 예산 소진 안 됨
  → Broad 타겟 + 연령만 조정하는 것이 소규모 예산에서 최적
```

### 왜 관심사를 안 넣나?
- 소규모 예산(₩10,000/일)에서 관심사 타겟팅을 하면 오디언스가 너무 작아짐
- Meta 알고리즘이 "인테리어 관심" 외에도 전환 가능한 사용자를 놓침
- Broad 타겟이 최근 Meta에서 권장하는 방식 (Advantage+ 기반)

## 랜딩 URL UTM 규칙

### 형식
```
도메인/form?utm_source={플랫폼}&utm_medium={배치}&utm_content={소재코드}&utm_campaign={캠페인}
```

### 예시
```
# 스토리 소재 S2
도메인/form?utm_source=instagram&utm_medium=story&utm_content=s2&utm_campaign=creative_test_v2

# 피드 소재 F5
도메인/form?utm_source=instagram&utm_medium=feed&utm_content=f5&utm_campaign=creative_test_v2

# 폼 A/B 테스트 시
도메인/form?...&utm_campaign=form_v1
도메인/form-v2?...&utm_campaign=form_v2
```

### 왜 UTM을 쓰나?
- 어떤 소재에서 어떤 폼으로 전환이 발생했는지 추적
- Google Analytics 또는 자체 DB에서 소재별 전환율 비교 가능
- AdLandingBanner가 `utm_source` 감지하여 광고 유입 시에만 배너 표시

## 성과 판단 기준 (7일 후)

### 소재 성과 비교 (Phase 1)

| 지표 | 좋음 | 보통 | 나쁨 |
|------|------|------|------|
| CTR (클릭률) | 3%+ | 1~3% | 1% 미만 |
| CPC (클릭당 비용) | ₩100 이하 | ₩100~300 | ₩300+ |
| 랜딩뷰 비용 | ₩200 이하 | ₩200~500 | ₩500+ |

### 승자 판정
```
1. CTR 기준 상위 3개 소재 선정
2. CPC 기준 상위 3개 소재 선정
3. 두 기준 모두에서 상위인 소재 = 승자
4. 7일간 노출 500회 이상인 소재만 유효 판정
```

### 폼 성과 비교 (Phase 2)

| 지표 | 계산 | 목표 |
|------|------|------|
| 폼 진입률 | 랜딩뷰 중 폼 시작한 비율 | 30%+ |
| 폼 완료율 | 폼 시작 중 제출 완료 비율 | 10%+ |
| 전체 전환율 | 랜딩뷰 대비 제출 완료 | 3%+ |

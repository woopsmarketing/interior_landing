  있으면 좋을 서브에이전트들을 한번 쭈욱 정리해줄래?
  출물 기준

  예:

  PRD 문서

  API 명세

  DB schema

  UI 컴포넌트

  테스트 코드

  디버그 리포트

  배포 체크리스트

  4) 호출 시점 기준

  예:

  작업 시작 전에만 호출

  구현 중간에 반복 호출

  에러 발생 시에만 호출

  배포 직전에만 호출

  이 기준으로 나누면 에이전트가 안 겹치고, 재사용이 잘 된다.

  이런식으로 나는 아주 많은 서브에이전트가 필요하거든?
  진짜 커다란 회사의 개발 팀 처럼 각각 업무를 쥐어주면서 나는 아주 큰 규모의 개발팀 하나를 만들고 싶은 그런 개념이야.
  이렇게 아주 체계적이고 구조적인 팀을 하나 만든다고 가정했을때,
  나에게 필요한 서브에이전트 구조 및 오케스트라 하나를 subagent_tutorial.md 파일로 하나 만들어줘.

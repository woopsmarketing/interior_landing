아니 내말은 내가 만약에 서브에이전트들을 다 등록하면 앞으로 어떻게 사용하면 되는지가 핵심이야.
  prd파일을 내가 직접 만들어야한다고했잖아?
  내가 클로드코드에서 프롬프트 요청을 전달하면 prd를 만들어주는 서브에이전트가 있으면 되지않을까?
  내가 직접 prd를 작성하기에는 너무 무리가 있는것같아서말이야.
  prompt-refiner 가 해주는 역할이먼데?
  출력을 보니까 5요소 공식으로 정제된 명령문 (파일 저장 없이 Orchestrator 에 바로 전달) 이라고
  나오고있는데, 내가 구현하려고하는 기능이나 그런것들을 최대한 구체적으로 적어서 클로드코드에게 프롬프트   
  전달하겠지만, 부족한 부분이있을수 있으니 그부분을 보완해주는게 prompt-refiner 잖아? 그리고나서
  prompt-refiner가 5요소 형식으로 정제해서, Orchestrator 에게 전달해주는건데 하다보면 prd-analyzer 가      
  PRD문서를 읽고 구현 가능한 태스크로 분해 한다고했는데, 여기서 말하는 prd문서가 무엇인지 말하는거야.      
  나는 prd문서를 매번 만들지는 않을거거든. 내가 prd문서를 잘 작성하는 방법도 모르고말이야.
  그래서prompt-refiner 가 PRD 문서도 같이 만들어줘야하지 않나 싶은거야.
  만든 prd문서를 기반으로 prd-analyzer 가 분석하고 구현가능한 태스크로 분해해주는것이지.

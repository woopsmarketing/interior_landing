# 인테리어 견적 요청 폼 기획 정리 v2

## 목적

이 문서는 한국형 인테리어 견적 요청 폼을 실제 서비스에 적용하기 위한 구조와 입력 항목을 정리한 문서입니다.

이번 버전의 핵심 방향은 아래와 같습니다.

- 전체 견적 요청 과정을 5~6단계 이내로 구성
- 개인정보 입력 및 개인정보 수집 동의는 반드시 마지막 단계에서 진행
- 현재 공간 이미지와 원하는 감성 이미지를 업로드받아 AI 예상 인테리어 결과 이미지 생성에 활용
- 필수 항목은 최소화하고, 선택 항목은 접기/펼치기 구조로 구성
- 방문자가 부담 없이 끝까지 작성할 수 있도록 이탈률 최소화

---

# 한 줄 결론

**이탈을 줄이려면 필수 입력은 앞단에서 최소한으로 받고, 선택 입력은 접기/펼치기로 숨기며, 개인정보는 마지막 단계에서만 받는 구조가 가장 좋습니다.**

---

# 추천 최종 프로세스

## 전체 단계 수
- 5단계 또는 6단계

## 가장 추천하는 구조
1. 공간 기본 정보
2. 공사 희망 정보
3. 취향 및 우선순위
4. 이미지 업로드 및 추가 요청사항
5. 개인정보 입력 및 동의
6. 제출 완료

실제 입력 단계는 5단계로 보고, 마지막 완료 화면까지 포함하면 6단계로 보면 됩니다.

---

### 필수 항목
- 공간 유형
- 지역
- 면적

### 선택 항목
- 상세 주소
- 건물명 / 상호명
- 전용면적
- 공급면적
- 평수
- 현재 상태
  - 공실
  - 거주 중
  - 운영 중
  - 입주 전
- 노후도
  - 신축
  - 부분 노후
  - 전체 노후
  - 잘 모르겠음

### 공간 유형 예시
- 아파트
- 빌라 / 연립
- 단독주택
- 오피스텔
- 사무실
- 상가
- 카페
- 음식점

- 학원
- 미용실

## STEP 2. 공사 희망 정보

견적의 핵심이 되는 공사 범위, 시기, 예산을 받는 단계입니다.

### 필수 항목
- 공사 범위
- 공사 희망 시기
- 예산 범위

### 선택 항목
- 공사 목적
  - 첫 입주
  - 이사 전 리모델링
  - 노후 개선
  - 매장 오픈
  - 브랜드 리뉴얼
  - 운영 효율 개선
  - 임대용 세팅
  - 기타
- 희망 시작일
- 희망 완료일
- 입주 예정일 / 오픈 예정일
- 일정 유연성
  - 날짜 고정
  - 1주 조정 가능
  - 2주 이상 조정 가능
- 공사 중 사용 여부
  - 비워둘 예정
  - 일부 거주

### 공사 범위 예시

- 2,000만원 ~ 3,000만원

- 5,000만원 이상
- 미정 / 상담 후 결정

### UI 전략
- 공사 범위 / 시기 / 예산만 기본 노출
- 세부 일정은 접기/펼치기 구조
- “예산 미정”도 선택 가능하게 구성

## STEP 3. 취향 및 우선순위

이 단계는 사용자가 원하는 방향을 빠르게 파악하기 위한 구간입니다.

### 필수 항목
- 중요하게 생각하는 요소 1개 이상 선택

### 선택 항목
- 선호 스타일
- 선호 색감
- 선호 분위기
- 현재 공간에서 불편한 점
- 라이프스타일 / 사용 목적

### 중요하게 생각하는 요소 예시
중복선택 가능

- 가성비
- 디자인
- 편의성
- 수납력
- 내구성
- 청소 편의성
- 채광
- 조명 분위기
- 방음
- 친환경 자재
- 반려동물 친화
- 아이 안전
- 공사 기간
- 유지관리
- 공간 활용도

### 선호 스타일 예시
- 미니멀
- 모던
- 내추럴
- 북유럽
- 호텔식
- 럭셔리
- 인더스트리얼
- 빈티지
- 한옥 감성
- 잘 모르겠음

### 선호 분위기 예시
- 밝고 깨끗한 분위기
- 따뜻하고 아늑한 분위기
- 세련되고 도시적인 분위기
- 고급스럽고 묵직한 분위기
- 감성적이고 부드러운 분위기
- 오래 질리지 않는 분위기

### 현재 공간의 불편한 점 예시
중복선택 가능

- 수납이 부족함
- 좁아 보임
- 동선이 불편함
- 조명이 부족함
- 주방이 불편함

- 청소가 불편함
- 소음 문제가 있음
- 콘센트가 부족함
- 공간 분리가 안 됨

### UI 전략
- “가장 중요하게 생각하는 요소”만 기본 노출
- 나머지 스타일 / 불편사항 / 라이프스타일은 선택 입력으로 펼치기
- 선택지가 많더라도 체크박스 위주로 구성해서 입력 피로 줄이기

## STEP 4. 이미지 업로드 및 추가 요청사항

이 단계는 AI 예상 결과 이미지를 만들기 위한 핵심 단계입니다.

### 필수 항목
- 없음

### 선택 항목
- 현재 공간 이미지 업로드
- 원하는 결과물의 감성 이미지 업로드
- 추가 요청사항

### 업로드 항목 설명 예시
#### 현재 공간 이미지 업로드(선택)
- 현재 공간 사진을 올려주시면 공간 상태를 더 정확히 파악할 수 있습니다.

#### 원하는 결과물의 감성 이미지 업로드(선택)
- 원하는 인테리어 분위기와 비슷한 참고 이미지를 올려주시면 스타일 제안과 AI 예상 이미지 생성에 도움이 됩니다.

### 추가 요청사항(선택)
자유 입력란은 꼭 넣는 것이 좋습니다.

#### 추천 라벨
- 추가 요청사항(선택)

#### 추천 플레이스홀더
- 원하는 분위기, 꼭 반영하고 싶은 요소, 피하고 싶은 스타일, 필요한 공간 기능 등을 자세히 작성해주시면 더욱 정확한 상담과 AI 제안에 도움이 됩니다.

### UI 전략
- 이미지 업로드는 2개 박스를 직관적으로 구분
- “사진 없이도 진행 가능” 문구 삽입
- 추가 요청사항은 기본 노출하되 선택 항목으로 표시
- 업로드 이미지가 많을수록 더 정확한 결과를 받을 수 있다는 보상형 문구 제공

## STEP 5. 개인정보 입력 및 동의

개인정보 관련 단계는 반드시 마지막에 배치합니다.

### 필수 항목
- 이름
- 연락처
- 개인정보 수집 및 이용 동의
- 상담 진행을 위한 연락 수신 동의

### 선택 항목
- 이메일
- 상담 희망 방식
  - 전화
  - 문자
  - 카카오톡
  - 이메일
- 연락 가능 시간대
  - 오전
  - 오후
  - 저녁
- 이벤트 / 프로모션 / 포트폴리오 안내 수신 동의(선택)

### UI 전략
- 마지막 단계에서만 개인정보를 받음
- 앞 단계까지는 최대한 부담 없이 작성 가능하도록 설계
- 개인정보를 입력하면 상담 신청이 완료된다는 흐름을 명확하게 보여주기
- 동의 체크박스 문구는 최대한 짧고 명확하게 작성

# 접기 / 펼치기 구조 추천

## 기본 원칙
- 방문자가 처음 보는 화면에는 필수 항목만 보이게
- 선택 항목은 “더 자세히 입력하기” 버튼으로 펼치기
- 단계마다 선택 입력은 최대 1개 섹션만 펼치도록 구성

## 예시 문구
- 추가 공간 정보 입력하기
- 세부 일정 입력하기
- 스타일 취향 더 선택하기
- 이미지와 상세 요청 더 입력하기

이 방식이 가장 부담이 적습니다.

---

# 추천 안내 문구

## 폼 상단 문구
필수 항목만 입력하셔도 상담 신청이 가능합니다.  
추가 정보를 작성해주실수록 더 정확한 견적과 AI 인테리어 제안을 받을 수 있습니다.

## 이미지 업로드 안내 문구
현재 공간 이미지와 원하는 감성 이미지를 업로드해주시면,  
이를 바탕으로 예상 인테리어 결과 이미지를 AI로 제안해드릴 수 있습니다.

## 추가 요청사항 안내 문구
상세하게 작성해주실수록 상담 품질이 높아지고, 원하는 방향을 더 정확하게 반영할 수 있습니다.

# 추천 완료 흐름

## 완료 화면 예시
- 견적 요청이 접수되었습니다.
- 입력해주신 정보와 이미지를 바탕으로 상담 및 AI 제안 준비를 진행합니다.
- 작성해주신 연락처로 순차적으로 안내드릴 예정입니다.

# 최종 정리

## 가장 추천하는 구조
- 총 5단계 입력 + 완료 화면
- 개인정보는 마지막 단계
- 이미지 업로드는 중간 단계
- 필수 입력은 최소화
- 선택 입력은 접기/펼치기
- 자유 입력란에는 상세 작성 유도 플레이스홀더 삽입

## 핵심 포인트
사용자는  
“생각보다 간단하게 신청할 수 있다”는 느낌을 받아야 하고,

운영자는  
“자세히 입력한 사용자의 정보는 충분히 받을 수 있어야” 합니다.

이 구조가 가장 실용적이고 이탈률 관리에도 유리합니다.

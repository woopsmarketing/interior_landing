# AI 인테리어 견적 비교 랜딩페이지 PRD

- 문서 버전: v1.0
- 작성일: 2026-03-09
- 문서 목적: 광고 유입용 랜딩페이지의 정보 구조, 섹션 설계, 카피 방향, UX 흐름, 폼 구조, 핵심 메시지를 하나의 기준 문서로 정리한다.
- 산출물 범위: 랜딩페이지 전체 섹션 구조, CTA 이후 폼 구조, 섹션별 목적/콘텐츠/카피/레이아웃/주의사항, 전환 포인트, 필수 고지 문구

## 1. 프로젝트 개요

### 1.1 프로젝트명
AI 인테리어 견적 비교 랜딩페이지

### 1.2 배경
정식 인테리어 플랫폼 구축 전에 광고를 통해 실제 수요와 전환 가능성을 검증한다.  
단순한 "업체 비교"가 아니라, 사용자가 느끼는 막막함을 줄이고 **사진·요구사항·예산·원하는 무드**를 정리해서 업체 견적 비교까지 이어지는 흐름을 검증하는 것이 목적이다.

### 1.3 핵심 가치 제안
이 서비스는 인테리어를 하고 싶지만 아래 문제로 시작하지 못하는 사용자에게 적합하다.

- 어디서부터 시작해야 할지 모르겠다.
- 원하는 스타일은 있는데 업체에 설명하기 어렵다.
- 견적을 받아도 무엇을 비교해야 하는지 모르겠다.
- 업체를 잘못 선택할까 봐 불안하다.
- 같은 설명을 여러 업체에 반복하는 것이 귀찮다.

이 랜딩페이지는 이런 문제를 아래 방식으로 해결한다.

- 고객의 조건을 먼저 정리한다.
- 필요한 경우 사진과 참고 이미지를 받아 더 구체화한다.
- 사진 첨부 시 AI 예시 이미지 생성 가능성을 제공한다.
- 전국 100개 이상 등록 인테리어 업체에 한 번의 요청으로 견적 기회를 만든다.
- 고객은 정리된 니즈 + 견적 결과를 보고 원하는 업체를 선택한다.

### 1.4 목표

#### 비즈니스 목표
- 광고 유입의 전환율 검증
- 고객 문의 데이터 수집
- 실제 수요 확인
- 추후 정식 플랫폼 개발을 위한 요구사항 검증

#### 사용자 목표
- 어렵지 않게 인테리어 문의를 시작한다.
- 원하는 내용을 정리해서 전달한다.
- 여러 업체 견적을 편하게 비교한다.
- 필요 시 예시 이미지로 방향성을 더 구체적으로 본다.

## 2. 타겟 사용자 정의

### 2.1 핵심 타겟
- 인테리어 초보 고객
- 아파트/주택/오피스/상업공간 리모델링을 고민하는 고객
- 예산은 정해져 있지만 방향 정리가 안 된 고객
- 업체를 하나씩 알아보는 과정이 번거로운 고객
- 원하는 감성은 있으나 전문 용어로 설명하기 어려운 고객

### 2.2 고객의 주요 감정
- 기대감보다 막막함이 먼저 든다.
- 돈이 많이 드는 결정이라 쉽게 시작하지 못한다.
- 업체 선택 실패가 두렵다.
- 추가 비용이 생길까 걱정된다.
- 내 취향이 제대로 반영되지 않을까 걱정된다.

### 2.3 고객이 중요하게 보는 것
- 견적의 투명성
- 내 취향 반영 가능성
- 업체 신뢰도
- 비교 편의성
- 공사 범위와 비용의 명확성
- 부담 없이 시작할 수 있는지 여부

## 3. 랜딩페이지 핵심 메시지 구조

### 3.1 메인 메시지
**인테리어를 하고 싶은 마음은 분명한데, 어디서부터 시작해야 할지 막막한 사람을 위해, 원하는 조건을 먼저 정리하고 여러 업체 견적 비교까지 쉽게 시작할 수 있게 돕는 서비스**

### 3.2 보조 메시지
- 상담 신청은 무료다.
- 원하는 업체는 고객이 직접 선택할 수 있다.
- 한 번의 요청으로 여러 업체 견적을 받을 수 있다.
- 사진은 필수가 아니지만, 첨부하면 더 자세한 요청 정리와 AI 예시 이미지에 도움이 된다.
- AI 이미지는 방향을 이해하기 위한 예시 이미지이며 실제 시공 결과와 다를 수 있다.

### 3.3 절대 놓치면 안 되는 메시지
- “어렵지 않다”
- “혼자 헤매지 않아도 된다”
- “여러 업체를 비교할 수 있다”
- “최종 선택은 고객이 직접 한다”
- “사진은 선택사항이다”

## 4. UX 원칙

### 4.1 핵심 원칙
1. 첫 화면에서 공감이 먼저 나와야 한다.
2. 기능 설명보다 감정 해소가 먼저 와야 한다.
3. 폼 진입 장벽은 최대한 낮아야 한다.
4. AI는 주인공이 아니라 보조 수단으로 보이게 해야 한다.
5. 사진 첨부가 필수처럼 느껴지면 안 된다.
6. 강요 없이 비교하고 선택할 수 있는 구조여야 한다.

### 4.2 톤앤매너
- 감정 공감형
- 따뜻하지만 과하지 않음
- 쉬운 언어 사용
- 전문가처럼 보여도 어렵게 느껴지지 않음
- 광고 랜딩페이지 특성상 한 문장 한 문장이 명확해야 함

## 7. 섹션별 상세 PRD

# Section 0. Header

## 목적
- 상단에서 신뢰감 제공
- 언제든 CTA 클릭 가능하게 함

## 권장 구조
- 좌측: 로고
- 우측: 짧은 신뢰 문구 1~2개 + CTA 버튼

## 추천 요소
- 상담 신청 무료
- 여러 업체 비교 가능
- [무료로 견적 요청하기]

## UI 가이드
- 스티키 헤더 권장
- 메뉴는 최소화
- 네비게이션 링크를 많이 넣지 않는다

## 주의사항
- 상단에 너무 많은 메뉴를 두면 이탈이 늘어날 수 있음
- 랜딩페이지 목적은 탐색이 아니라 전환임

# Section 1. Hero

## 목적
- 방문자가 "내 이야기다"라고 느끼게 만든다.
- 서비스가 무엇인지 즉시 이해시키고 CTA까지 연결한다.

## 추천 구조
1. 메인 헤드라인
2. 서브카피
3. CTA 버튼 2개
4. 보조 신뢰 문구
5. 공감 포인트 3개

## 확정 카피
### 메인 헤드라인
**인테리어를 바꾸고 싶은 마음은 분명한데, 어디서부터 시작해야 할지 막막하셨나요?**

### 서브카피
예산은 한정적이고, 원하는 분위기는 있는데, 어떤 업체를 골라야 할지 모르겠다면  
이제 혼자 헤매지 마세요.  
당신의 취향과 공간 조건, 예산을 먼저 정리하고, 그에 맞는 인테리어 업체를 비교할 수 있도록 도와드립니다.

### CTA
- **무료로 내 조건 전달하기**
- **나에게 맞는 업체 알아보기**

### 보조 문구
- 상담 신청 무료
- 원하는 업체 직접 선택 가능
- 여러 업체 비교 가능

### 공감 포인트
- 원하는 스타일은 있는데, 어떻게 설명해야 할지 모르겠는 분
- 여러 견적을 받아도 무엇을 비교해야 하는지 헷갈리는 분
- 공사비보다 더 무서운 추가금과 업체 선택 실패가 걱정되는 분

## 레이아웃 제안
- 좌측: 카피 + CTA
- 우측: 간단한 폼 미리보기 카드 또는 요청 입력 예시 카드

## 오른쪽 카드 예시
```text
원하는 스타일: 미니멀 / 우드톤 / 따뜻한 분위기
공간: 24평 거실 + 주방
예산: 2,000만 원대
중요 포인트: 수납 / 관리 편의 / 동선
```

## 주의사항
- Hero에서 AI 기능을 전면에 크게 띄우지 않는다.
- Hero는 어디까지나 공감과 쉬운 시작이 중심이다.

# Section 2. Problem / Empathy

## 목적
- 사용자가 느끼는 문제를 명확히 언어화한다.
- “나만 어려운 게 아니구나”라는 심리를 만든다.

## 권장 구조
1. 섹션 제목
2. 공감 본문
3. 문제 카드 4개

## 카피
### 제목
**인테리어가 어려운 이유는, 하고 싶은 마음보다 걱정이 먼저 앞서기 때문입니다**

### 본문
인테리어는 누구나 기대를 안고 시작하지만,  
막상 준비하려고 하면 생각보다 더 많은 고민이 생깁니다.

원하는 분위기는 있는데 어떻게 설명해야 할지 모르겠고,  
견적은 받아도 왜 이렇게 차이가 나는지 이해하기 어렵고,  
업체를 고르는 순간에는 더 신중해질 수밖에 없습니다.

그래서 많은 분들이  
인테리어를 하고 싶어도 쉽게 시작하지 못합니다.

### 문제 카드
#### 카드 1
**견적이 제각각이라 비교가 어렵습니다**  
같은 공간인데도 업체마다 금액과 설명이 달라 판단이 쉽지 않습니다.

#### 카드 2
**내 취향을 말로 설명하기 어렵습니다**  
원하는 느낌은 분명한데, 전문가처럼 정리해서 전달하기는 어렵습니다.

#### 카드 3
**업체 선택이 가장 불안합니다**  
정말 잘 맞는 업체인지, 믿고 맡겨도 되는지 고민됩니다.

#### 카드 4
**상담할수록 오히려 더 헷갈립니다**  
같은 이야기를 반복할수록 피로는 쌓이고 비교는 더 어려워집니다.

## UI 가이드
- 2x2 카드 레이아웃 권장
- 아이콘은 단순하게
- 텍스트는 짧고 명확하게

# Section 3. Core Value

## 목적
- 일반적인 견적 비교 서비스와 다른 점을 설명한다.
- 한 번의 요청으로 여러 업체 비교가 가능하다는 장점을 명확히 전달한다.

## 권장 구조
1. 제목
2. 설명 본문
3. 핵심 가치 3포인트

# Section 4. How It Works

## 목적
- 서비스 이용이 복잡하지 않다고 느끼게 만든다.
- 사진이 선택사항임을 분명히 한다.
- AI 예시 이미지가 자연스럽게 포함되도록 한다.

## 권장 구조
1. 제목
2. 4단계 프로세스
3. 하단 안내 문구

## 카피
### 제목
**복잡하게 알아보지 않아도, 아래 순서대로 쉽게 시작할 수 있습니다**

### Step 1
**원하는 인테리어를 작성합니다**  
원하는 분위기, 예산, 시공 범위, 중요하게 생각하는 조건 등을 남겨주세요.  
아직 완벽하게 정리되지 않았어도 괜찮습니다.

**공간 사진과 참고 이미지는 선택사항입니다.**  
다만 함께 첨부해주시면 더 구체적인 요청 정리와 예시 이미지 제공에 도움이 됩니다.

### Step 2
**요청 내용을 더 이해하기 쉽게 정리합니다**  
고객이 남긴 내용을 바탕으로  
원하는 인테리어 방향과 핵심 니즈를 더 보기 쉽게 정리합니다.

공간 사진과 참고 이미지를 함께 제출한 경우에는  
이를 바탕으로 **AI 예시 이미지**도 함께 제공될 수 있습니다.

### Step 3
**전국 인테리어 업체가 내용을 검토합니다**  
정리된 요청 내용을 바탕으로  
전국 100개 이상의 등록 인테리어 업체가 고객의 요청을 검토하고  
견적과 작업 방향을 제안합니다.

### Step 4
**견적 결과를 비교하고 원하는 업체를 선택합니다**  
여러 업체의 견적과 제안을 비교한 뒤  
고객이 원하는 업체를 직접 선택할 수 있습니다.

### 안내 문구
**제공되는 이미지는 AI가 생성한 예시 이미지이며, 실제 시공 결과와는 차이가 있을 수 있습니다.**

## 시각화 제안
- 1→2→3→4 단계형 플로우
- 데스크톱은 가로형, 모바일은 세로형

```text
[내용 작성]
   ↓
[요청 정리 + 선택 시 AI 예시]
   ↓
[업체 검토 및 견적 제안]
   ↓
[비교 후 직접 선택]
```

## 주의사항
- “사진이 필수”처럼 보이는 문구 금지
- AI 이미지를 과장된 결과 보장처럼 보이게 하면 안 됨

## 8. CTA 클릭 후 폼 PRD

### 8.1 목적
- CTA 클릭 직후 사용자의 행동 흐름이 끊기지 않도록 한다.
- AI 관련 안내는 랜딩페이지 본문이 아니라 폼 내부에서 자연스럽게 설명한다.

### 8.2 폼 오픈 방식
- 인라인 렌더링 또는 모달 렌더링
- 권장: **인라인 멀티스텝 폼**

### 8.3 멀티스텝 권장 이유
- 한 번에 입력 항목이 많으면 이탈률 증가
- 단계별 입력으로 부담 감소
- 진행률 표시가 가능함
- 광고 유입 사용자에게 더 적합함

### 8.4 폼 구조

#### Step 1. 기본 정보
- 이름
- 연락처
- 지역
- 공간 유형

#### Step 2. 원하는 인테리어 정보
- 예산
- 시공 범위
- 원하는 스타일
- 중요하게 생각하는 요소
- 상세 요청사항

#### Step 3. 선택 업로드
- 공간 사진 업로드 (선택)
- 참고 이미지 업로드 (선택)

##### 업로드 안내 문구
사진을 첨부해주시면 요청 내용을 더 구체적으로 정리할 수 있으며,  
AI 예시 이미지 제공에도 도움이 됩니다.

##### AI 안내 문구
제공되는 이미지는 AI가 생성한 예시 이미지이며,  
실제 시공 결과와는 차이가 있을 수 있습니다.

#### Step 4. 동의 및 제출
- 개인정보 동의
- 상담/견적 요청 동의
- 제출 버튼

### 8.5 폼 UX 가이드
- 진행률 표시: 예) 1/4, 2/4
- 업로드 영역은 필수 표시 금지
- 텍스트 플레이스홀더 예시 제공
- 모바일 우선 설계

### 8.6 폼 제출 후 메시지
#### 완료 제목
**요청이 접수되었습니다**

#### 완료 본문
입력해주신 내용을 바탕으로 요청 사항을 정리한 뒤,  
업체 검토 및 견적 진행에 활용할 예정입니다.

공간 사진과 참고 이미지를 첨부해주신 경우에는

## 13. 최종 요약

이 랜딩페이지는 단순히 인테리어 업체를 보여주는 페이지가 아니다.  
핵심은 **막막한 고객이 쉽게 시작하도록 만들고, 자신의 조건을 정리해서 여러 업체 견적을 비교할 수 있도록 돕는 것**이다.

페이지 구조는 아래 흐름을 반드시 유지해야 한다.

**공감 → 문제 인식 → 핵심 가치 → 진행 방식 → 대상 고객 → 기대 결과 → 안심 요소 → FAQ → 최종 CTA**

그리고 AI 이미지는 별도 대주제가 아니라,  
**폼 단계에서 자연스럽게 안내되는 보조 기능**으로 다루는 것이 가장 자연스럽다.

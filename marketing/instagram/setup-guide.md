# 인스타그램 광고 설정 가이드

> AI 인테리어 견적 비교 서비스 — 랜딩페이지 전환(리드) 캠페인

---

## 1. 메타 비즈니스 계정 생성

1. [business.facebook.com](https://business.facebook.com) 접속
2. **계정 만들기** 클릭
3. 비즈니스 이름, 본인 이름, 이메일 입력
4. 비즈니스 정보 입력 (업종: 주거 서비스 / 인테리어)
5. 이메일 인증 완료

> 이미 개인 페이스북 계정이 있어야 비즈니스 계정을 만들 수 있다.

---

## 2. 인스타그램 프로페셔널 계정 전환 + 페이스북 페이지 연결

### 인스타그램 프로페셔널 계정 전환

1. 인스타그램 앱 → **설정 및 활동** → **계정 유형 및 도구**
2. **프로페셔널 계정으로 전환** 선택
3. 카테고리 선택: **주택 개량 서비스** 또는 **인테리어 디자인 서비스**
4. **비즈니스** 계정 유형 선택 (크리에이터 아님)
5. 연락처 정보 입력

### 페이스북 페이지 연결

1. 인스타그램 설정 → **계정 센터** → **프로필**
2. **페이스북 페이지 연결** 선택
3. 메타 비즈니스 스위트에서 만든 페이지와 연결
4. 연결 완료 확인

> 페이스북 페이지가 없으면 새로 생성해야 한다. 페이지 이름은 서비스명과 동일하게 설정한다.

---

## 3. 광고 관리자 접속

1. [adsmanager.facebook.com](https://adsmanager.facebook.com) 접속
2. 비즈니스 계정으로 로그인
3. 좌측 메뉴에서 **광고 관리자** 선택
4. 광고 계정이 자동 생성되어 있는지 확인
5. 없으면: 비즈니스 설정 → 광고 계정 → **추가** → **새 광고 계정 만들기**

---

## 4. 결제 수단 등록

1. 광고 관리자 → **설정** (좌측 하단 톱니바퀴)
2. **결제 설정** 클릭
3. **결제 수단 추가**
4. 선택 가능:
   - 신용/체크카드 (권장)
   - PayPal
5. 카드 정보 입력 후 저장
6. 결제 한도 설정 (선택): 월 예산 상한선 설정 권장

> 처음에는 소액부터 시작하므로 결제 한도를 설정해두면 예산 초과를 방지할 수 있다.

---

## 5. 캠페인 생성

1. 광고 관리자 → **+ 만들기** 버튼 클릭
2. **캠페인 목표** 선택:
   - **리드** 선택 (구 "전환" 목표)
   - 리드 생성 방법: **웹사이트** 선택 (랜딩페이지로 유도)
3. 캠페인 이름 설정: `인테리어견적_리드_YYYYMMDD`
4. **A/B 테스트**: 켜기 (소재별 비교 테스트)
5. **캠페인 예산 최적화(CBO)**: 초기에는 끄기 (광고 세트별 예산 관리)
6. **다음** 클릭

### 캠페인 구조 권장

```
캠페인: 인테리어견적_리드
├── 광고 세트 A: 코어 타겟 (25-45세)
│   ├── 광고 1: Before/After 소재
│   ├── 광고 2: 고민 공감형 소재
│   └── 광고 3: AI 미리보기 소재
├── 광고 세트 B: 리타겟팅
│   ├── 광고 1: 숫자 임팩트 소재
│   └── 광고 2: 후기 소재
└── 광고 세트 C: 유사 타겟
    ├── 광고 1: Before/After 소재
    └── 광고 2: 고민 공감형 소재
```

## 6. 광고 세트: 타겟 오디언스 설정

### 코어 타겟 (Cold Audience)

| 항목 | 설정값 |
|------|--------|
| 연령 | 25-45세 |
| 성별 | 전체 |
| 지역 | 대한민국 (또는 서비스 지역 한정) |
| 관심사 | 인테리어 디자인, 리모델링, 홈데코, 아파트, 부동산, 신혼, 이사, 집꾸미기 |
| 행동 | 최근 이사, 신혼부부, 주택 소유자 |

### 관심사 상세 설정

```
카테고리: 관심사 → 주거
- 인테리어 디자인
- 리모델링
- 홈 인테리어
- DIY 홈 인테리어

카테고리: 관심사 → 부동산
- 부동산
- 아파트
- 주택 구매

카테고리: 라이프 이벤트
- 최근 이사함
- 신혼
- 약혼
```

### 확장 타겟: 유사 타겟 (Lookalike)

> 전환 데이터가 100건 이상 쌓인 후 생성 가능

1. 광고 관리자 → **타겟** → **유사 타겟 만들기**
2. 소스: 랜딩페이지 전환자 (폼 제출 완료자)
3. 유사도:
   - **1%**: 가장 유사 (소규모, 높은 전환율 기대)
   - **1-3%**: 중간 범위 (권장 시작점)
   - **3-5%**: 넓은 범위 (도달 확대 시)
4. 지역: 대한민국

### 리타겟팅 (Retargeting)

> Meta Pixel 설치 필수 (tracking/setup-guide.md 참고)

| 타겟 | 설명 | 기간 |
|------|------|------|
| 랜딩페이지 방문자 중 미전환자 | 페이지 방문했으나 폼 미제출 | 최근 7일 |
| 폼 시작 후 이탈자 | 폼을 열었으나 완료하지 않음 | 최근 14일 |
| 랜딩페이지 재방문자 | 2회 이상 방문한 고관여 유저 | 최근 30일 |

리타겟팅 커스텀 타겟 생성:
1. 광고 관리자 → **타겟** → **커스텀 타겟 만들기**
2. 소스: **웹사이트**
3. 이벤트: `PageView` (방문자) 중 `Lead` 이벤트 제외 (미전환자)
4. 기간: 7일 (최근 방문자 우선)

## 7. 노출 위치 설정

### 권장 노출 위치

| 위치 | 우선순위 | 비고 |
|------|----------|------|
| Instagram 피드 | 필수 | 가장 높은 전환율 |
| Instagram 릴스 | 필수 | 높은 도달률, 낮은 CPM |
| Instagram 스토리 | 필수 | 몰입도 높음 |
| Instagram 탐색 탭 | 선택 | 신규 유저 발굴 |
| Facebook 피드 | 선택 | 추가 도달 (30대 이상 효과적) |

### 설정 방법

1. 광고 세트 → **노출 위치**
2. **수동 노출 위치** 선택 (자동 아님)
3. 플랫폼: **Instagram** 체크
4. 노출 위치: 피드, 릴스, 스토리 체크
5. Facebook은 초기에 제외하고 Instagram에 집중

> 초기에는 Instagram에 집중하여 데이터를 모은 후, 성과 분석 후 Facebook으로 확장한다.

## 8. 예산 설정

### 초기 테스트 예산 (1-2주)

| 항목 | 일 예산 | 월 환산 |
|------|---------|---------|
| 코어 타겟 광고 세트 | 1-2만원 | 30-60만원 |
| 리타겟팅 광고 세트 | 5천-1만원 | 15-30만원 |
| **합계** | **1.5-3만원** | **45-90만원** |

### 예산 운영 원칙

- **테스트 기간**: 최소 7일 (학습 기간 필요)
- **소재당 최소 예산**: 일 5천원 이상 (데이터 축적을 위해)
- **성과 기반 확장**: CPA(전환당 비용)가 목표 이하이면 예산 20-30%씩 증액
- **학습 단계 주의**: 광고 세트 수정 시 학습 단계가 초기화되므로 빈번한 수정 금지
- **요일별 차이**: 주말에 인테리어 관심이 높으므로 금-일 예산 증액 고려

### KPI 목표 (초기 기준)

| 지표 | 목표값 | 설명 |
|------|--------|------|
| CPC (클릭당 비용) | 300-800원 | 랜딩페이지 클릭 |
| CTR (클릭률) | 1.5% 이상 | 소재 효과 판단 기준 |
| CPA (전환당 비용) | 5,000-15,000원 | 폼 제출 완료 기준 |
| CVR (전환율) | 5-15% | 랜딩페이지 전환율 |

## 9. Meta Pixel 설치 확인

> 상세 설치 가이드는 `tracking/setup-guide.md` 참고

### 필수 확인 사항

1. **Pixel ID**가 랜딩페이지에 설치되어 있는지 확인
2. **Meta Pixel Helper** (Chrome 확장)로 작동 확인
3. 필수 이벤트 설정:
   - `PageView`: 랜딩페이지 로드 시
   - `Lead`: 폼 제출 완료 시
   - `ViewContent`: 폼 열기 시 (선택)
4. 광고 관리자 → **이벤트 관리자**에서 이벤트 수신 확인
5. 전환 최적화 이벤트: `Lead`로 설정

### 전환 API (CAPI) 설정 (권장)

- 브라우저 쿠키 차단 대응을 위해 서버 사이드 전환 추적 권장
- Next.js API Route에서 Meta Conversions API 호출 구현

## 10. A/B 테스트 설정

### 테스트 구조

```
A/B 테스트 라운드 1 (1주차)
├── 소재 A: Before/After 비교 이미지
├── 소재 B: 고민 공감형 텍스트 카드
└── 소재 C: AI 미리보기 강조

→ 승자 소재 확정

A/B 테스트 라운드 2 (2주차)
├── 승자 소재 + 캡션 변형 A
├── 승자 소재 + 캡션 변형 B
└── 승자 소재 + CTA 변형

→ 최적 조합 확정
```

### A/B 테스트 원칙

- **한 번에 하나의 변수만** 테스트 (소재 or 캡션 or 타겟)
- **최소 테스트 기간**: 7일 또는 전환 50건 이상
- **통계적 유의성** 확인 후 판단 (광고 관리자에서 자동 표시)
- **성과 지표**: CPA 기준으로 승자 판단
- **패배 소재**: 즉시 중단하여 예산 낭비 방지

### 설정 방법

1. 캠페인 만들기 → **A/B 테스트** 활성화
2. 테스트 변수: **크리에이티브** 선택
3. 동일 타겟/예산으로 소재만 다르게 설정
4. 테스트 기간: 7일
5. 승리 기준: **전환당 비용(CPA)** 선택

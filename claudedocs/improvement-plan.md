# 개선 제안 — AI 인테리어 견적 비교 랜딩페이지

> QA 리포트(`qa-report.md`) + 랜딩페이지 전환 노하우(`tip_landing.md`) 교차 분석
> 작성일: 2026-03-09

---

## 핵심 진단 요약

`tip_landing.md`에 따르면 전환하는 랜딩페이지가 반드시 답해야 하는 질문 3가지:

| 질문 | 현재 상태 | 평가 |
|------|-----------|------|
| ① 이 서비스가 내 문제를 해결해주나? | Hero + Problem 섹션 | ✅ 양호 |
| ② 어떻게 작동하고 결과는 어떻게 나오나? | HowItWorks + Outcomes | ⚠️ 프로세스 주체 불명확 |
| ③ 나같은 사람에게도 통하나? | WhoFor (텍스트만) | ❌ **소셜 증명 완전 없음** |

**가장 큰 구조적 공백**: ③번 질문에 답하는 소셜 증명(후기, 통계, 사례)이 전혀 없습니다.
tip_landing.md는 이 질문이 해소되지 않으면 이미 공감하고 있어도 전환이 일어나지 않는다고 명시합니다.

---

## 우선순위별 개선 항목

### 🔴 P1 — 전환율에 직접 영향 (즉시 수정)

#### 1. Form Step 1 필드 순서 변경

**문제**: 첫 화면에서 이름 + 전화번호를 요구 → 개인정보 선요구가 진입 장벽 최고 구간에서 발생

**개선 방향**:
```
현재: 이름 → 연락처 → 지역 → 공간 유형
변경: 지역 → 공간 유형 → (Step 2) 예산/스타일 → (마지막) 이름/연락처
```

**근거**: 인테리어 플랫폼에서 반복 검증된 패턴. 관심 정보 먼저 → 개인정보 나중. 첫 스텝 이탈 10–20% 개선 기대.

---

#### 2. Hero CTA 이중화 해소

**문제**: Primary("무료로 내 조건 전달하기") + Secondary("나에게 맞는 업체 알아보기")가 동일하게 폼 스크롤. 선택지 2개 → 결정 유예

**개선 방향**:
```
Primary: "무료로 내 조건 전달하기" → #form 스크롤 (유지)
Secondary: "어떻게 진행되나요?" → #how-it-works 앵커로 변경
```

**근거**: tip_landing.md — "아직 확신 없는 사용자는 탐색 경로로 유도해 이탈 방어." Secondary가 차별화된 역할을 가져야 결정 피로가 사라짐.

---

#### 3. WhoFor 하단 CTA (이미 적용됨, 효과 검증 필요)

**현황**: QA 이후 이미 추가됨
**다음 단계**: 실제 클릭률 측정 후 Trust, FAQ 하단에도 순차 추가

---

### 🟡 P2 — 전환 흐름 개선 (우선 적용 권장)

#### 4. 소셜 증명 섹션 신규 추가 ← **현재 완전 없음**

**문제**: tip_landing.md의 3번 질문("나같은 사람에게도 통하나?")에 WhoFor 텍스트만으로는 불충분. 실제 사례/수치가 없으면 공감 후에도 전환이 막힘.

**개선 방향**: HowItWorks와 WhoFor 사이에 추가

**옵션 A — 통계 배너** (데이터 확보 전까지 현실적)
```
예시 카드:
- 견적 요청 건수: 월 OOO건 (운영 데이터로 채울 것)
- 등록 업체 수: 100개+
- 평균 응답 시간: 영업일 기준 O일 이내
- 무료 신청: 0원
```

**옵션 B — 후기 카드** (서비스 런칭 후)
```
"처음엔 어디서 시작해야 할지 몰랐는데, 요청 하나로 3곳에서 견적이 왔어요."
— 30대 / 서울 / 24평 아파트 리모델링
```

**지금 할 수 있는 최소 버전**: 통계 배너 + "왜 우리를 선택하나요?" 형식의 수치 기반 섹션

---

#### 5. HowItWorks — Step 2 주체 명확화 + 인라인 CTA

**문제**: "요청 내용을 더 이해하기 쉽게 정리합니다"에서 누가 정리하는지 불명확 → 신뢰 손상 또는 AI에 대한 기대 오차

**개선 방향**:
```
현재: "요청 내용을 더 이해하기 쉽게 정리합니다"
변경: "서비스가 입력 내용을 읽기 쉬운 형식으로 정리합니다
      (공간 사진 첨부 시 AI 예시 이미지 생성 가능)"
```

**추가**: 섹션 하단에 텍스트 링크 CTA
```
지금 바로 시작하기 →   (→ #form 앵커)
```

---

#### 6. Trust 섹션 — 핵심 배지 강조 + CTA 추가

**문제**: 7개 배지가 flex-wrap 나열 → 모바일에서 순서 불규칙. "상담 신청 무료", "꼭 계약하지 않아도 됨" 같은 핵심이 묻힘

**개선 방향**:
```
상단: "상담 신청 무료" + "꼭 계약하지 않아도 됨" 2개를 크게 별도 강조
하단: 나머지 5개 배지 나열
맨 아래: CTA 버튼
```

---

#### 7. FAQ — AI 편중 해소 + 운영 FAQ 추가

**문제**: 6개 중 3개가 AI 이미지 관련. 실제 사용자 주요 의문 미반영

**추가 권장 항목**:
```
Q. 업체는 어떤 기준으로 등록되어 있나요?
A. 파트너 등록 과정을 거친 인테리어 업체만 참여할 수 있습니다.

Q. 견적은 어떻게 받게 되나요?
A. 요청 접수 후 영업일 기준 O일 이내 업체 담당자가 연락을 드립니다.
```

**FAQ 하단 CTA 추가**: "더 궁금한 점이 있으신가요?" + 버튼

---

#### 8. FinalCTA — 버튼 단일화

**문제**: 버튼 2개가 동일 동작 반복 (Hero와 동일한 문제)

**개선 방향**:
```
현재: "무료로 견적 요청 시작하기" + "내 조건 먼저 전달하기"
변경: "무료로 견적 요청 시작하기" 1개만 유지
     보조 문구: "사진 없이도 요청 가능 · 상담 신청 무료" (현재 유지)
```

---

### 🟢 P3 — 경험 품질 개선 (런칭 전 적용하면 좋음)

#### 9. 완료 화면 — 후속 프로세스 안내

**문제**: 제출 후 "요청이 접수되었습니다" + 짧은 설명만 있어 다음 단계 불안 해소 안 됨

**개선 방향**:
```
"요청이 접수되었습니다"

입력해 주신 내용을 바탕으로 요청 사항을 정리한 뒤,
업체 검토 및 견적 진행에 활용할 예정입니다.

[추가] 이렇게 진행됩니다:
  1. 요청 검토 (영업일 기준 1일 이내)
  2. 업체 매칭 및 견적 준비
  3. 담당자 연락 및 결과 전달

연락처로 안내 문자를 보내드렸습니다.
```

---

#### 10. Form Step 3 — 명시적 스킵 버튼

**문제**: "선택사항" 표기는 있지만 명시적 스킵 수단 없어 업로드 의무로 오인 가능

**개선 방향**:
```
[다음 단계로] 버튼 아래에
"사진 없이 계속하기 →" 텍스트 링크 추가
```

---

#### 11. Form Step 4 — 약관 링크

**문제**: 개인정보 동의 체크박스에 전문 링크 없음

**개선 방향**: "개인정보 수집·이용 동의 [자세히 보기]" 링크 추가 (모달 또는 새 탭)

---

#### 12. Problem 섹션 — 브리지 문구

**문제**: 4개 문제 카드 후 흐름이 끊김

**개선 방향**: 섹션 하단에 1줄 브리지 문구 추가
```
"이 문제들, 혼자 해결하지 않아도 됩니다."
```

---

#### 13. Outcomes 섹션 — Problem과 시각 차별화

**문제**: 동일한 2×2 카드 그리드 구조 반복 → "이미 본 것 같은" 피로감

**개선 방향**: 레이아웃 변경 (가로 스크롤 카드, 또는 좌우 2단 텍스트+아이콘 형식)

## A/B 테스트 우선순위

| 순위 | 테스트 항목 | 기대 효과 | 측정 지표 |
|------|------------|-----------|-----------|
| 1 | Step 1 필드 순서 (연락처 마지막) | 첫 스텝 이탈 10–20% 감소 | Step 1 완료율 |
| 2 | Hero Secondary CTA 탐색 경로 분리 | Primary CTA 클릭 집중 | CTA 클릭률 분리 측정 |
| 3 | 소셜 증명 섹션 유무 | 중간 구간 이탈 방어 | 폼 도달율 |

## 적용 순서 제안

```
즉시 (코드 수정 5분 이내)
├── Hero Secondary CTA 앵커 변경
├── FinalCTA 버튼 1개로 단일화
├── Form Step 3 "사진 없이 계속하기" 링크 추가
└── Problem 브리지 문구 1줄 추가

단기 (설계 필요)
├── Form Step 1 필드 순서 재설계
├── HowItWorks Step 2 카피 수정 + 하단 CTA
├── Trust 배지 강조 구조 변경 + CTA
└── FAQ 항목 2개 추가 + 하단 CTA

중기 (데이터/컨텐츠 확보 후)
├── 소셜 증명 섹션 신규 추가 (통계 or 후기)
├── 완료 화면 후속 안내 보강
├── Form Step 4 약관 링크
└── Outcomes 시각 차별화
```

## 유지할 강점 (건드리지 말 것)

- Hero 헤드라인/서브카피: PRD 공감 원칙에 정확히 부합
- "사진 없이도 요청 가능" 반복 메시지: 이탈 방어 효과적
- Step 4 "꼭 계약하지 않아도 됩니다" 강조 박스
- 전반적 톤앤매너 (친근하고 부담 없는 어조)
